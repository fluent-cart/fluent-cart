FluentCart Admin

Design tokens, components, and patterns for the admin panel.

Functional, not decorative.

Built for store managers processing orders, not visitors browsing a storefront. Every choice serves density, scannability, and speed.

Content-dense — More data, less whitespace. 72px rows, 20px padding, small badges.
Quiet chrome, loud signals — Neutral shell. Color only for status: green = paid, red = failed, yellow = pending.
One way to do it — One card, one badge, one table. Consistency over variety.
Dark mode is not optional — Every surface has a dark: counterpart. Required, not a feature.
Tokens, not values — No arbitrary hex or px. Everything from the token files.
Reuse, don't rebuild — Check Bits/Components/ first. It probably exists.

Color Intent

Text — 4 shades only: heading, body, secondary, muted
Surfaces — White cards, light gray backgrounds, slightly darker hover
Borders — Soft dividers, slightly stronger input outlines
Status — Success/warning/danger/blue. Badge = 100 bg + 800 text
Brand — Dark navy for logo, primary buttons, dark surfaces
Accent — Links and info highlights only. Never for buttons.
Data viz — 16 chart colors. Never in UI chrome.

Typography Intent

Semibold + xl — Page titles (one per page)
Medium + base — Card headers (section containers)
Medium + sm — Labels, table headers
Normal + sm — Body text, descriptions, cell content
Normal + xs — Meta: timestamps, hints, footers
If you need a size between these, you don't — pick the nearest one.
